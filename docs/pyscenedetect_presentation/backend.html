Backend: FastAPI e Celery para Processamento Assíncrono

Arquitetura de Processamento em Background

⚡
FastAPI
Framework web moderno para construir APIs REST de alta performance.
  • Endpoints para iniciar detecção
  • Consulta de status (task_id)
  • Resposta imediata ao cliente
⚙
Celery
Queue de tarefas distribuída para processamento em background.
  • Executa tarefas assincronamente
  • Gerencia status e progresso
  • Integra com Redis/RabbitMQ
Como Funciona a Integração
FastAPI e Celery trabalham juntos para criar um sistema robusto e escalável. FastAPI expõe os endpoints REST que o cliente chama, enquanto Celery gerencia a execução das tarefas pesadas em background.
Fluxo Assíncrono
1. Cliente → FastAPI
↓
2. FastAPI enfileira tarefa
↓
3. Retorna task_id imediatamente
↓
4. Celery processa em background
↓
5. Cliente consulta status
Benefícios
✓ Servidor não bloqueia durante processamento
✓ Suporta múltiplas tarefas simultâneas
✓ Escalável horizontalmente
✓ Recuperação automática de falhas
